# Claude Code Evolution Engine (CCEE)

> A self-contained research-and-upgrade pipeline that keeps a Claude Code VPS installation at the cutting edge.

**Status:** Phase 0 Bootstrap — ready for first run.

---

## What CCEE Does

CCEE runs weekly as a cron job and:

1. **Researches** latest Claude Code features, community frameworks (GSD, BMAD), and YouTube tutorials
2. **Audits** the current VPS Claude Code configuration against what it discovers
3. **Proposes** structured changes with risk ratings, rollback procedures, and validation commands
4. **Tests** proposals in a sandboxed copy of the config
5. **Presents** a review UI at `http://100.88.238.20:9898` for human approval
6. **Applies** accepted changes atomically, with rollback capability
7. **Documents** the result in Obsidian with SVG architecture diagrams

## Quick Start

```bash
# Clone and install
git clone https://github.com/gregdigittal/claude-code-evolution-engine.git
cd claude-code-evolution-engine
npm install

# Configure
cp .env.example .env
# Edit .env: set ANTHROPIC_API_KEY and GITHUB_TOKEN

# Scaffold Obsidian staging directory
./scripts/install.sh --scaffold-obsidian

# First full run
npm run ccee:run

# Install cron + systemd (optional)
./scripts/install.sh --cron --systemd
```

## Review UI

After a pipeline run, open the review UI to accept or reject proposals:

```
http://localhost:9898                        (local)
http://100.88.238.20:9898                   (Tailscale)
http://100.88.238.20:9898/ccee/review/YYYY-MM-DD
```

## Configuration

| Variable | Purpose | Required |
|----------|---------|----------|
| `ANTHROPIC_API_KEY` | Claude Code SDK auth | Yes |
| `GITHUB_TOKEN` | GitHub API for repo scanning | Yes |
| `OBSIDIAN_STAGING_PATH` | Path to Obsidian staging dir | No |
| `REVIEW_UI_PORT` | Express server port | No (default: 9898) |
| `VPS_TAILSCALE_IP` | Tailscale IP for review URL | No |
| `SLACK_WEBHOOK_URL` | Notifications webhook | No |

Full list in `.env.example`.

## Stack

- TypeScript 5.7, Node.js 20, ESM
- Express (Review UI)
- `@anthropic-ai/claude-code` SDK (LLM phases)
- `node-cron` (scheduling)
- Vitest (tests)
